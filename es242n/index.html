Times are IST. 24 June 2026: Graphs I Vocabulary first — vertex, edge, neighbour, degree, path, cycle, connected — then the three-function Graph interface (vertex_count, neighbors, has_edge) that BFS and DFS will call. Adjacency matrix vs adjacency list: same interface, different storage, different cost. BFS and DFS animated step-by-step with explicit visited / current / neighbour state; both O(V+E), only the visiting order differs. Closes with a coding exercise where students implement both backings in C and run BFS against either — same answer, different cost. The supplementary heaps-with-pointers note runs as a 15-minute warm-up. 22 June 2026: Heaps Opens by closing out binary search trees — the two ways to walk a tree: depth-first (the traversals already written — inorder, validate, LCA, delete) and breadth-first (level order with a queue) — and the one idea graphs will add next, a visited set. Then the day's topic: the heap. A heap is a binary tree with two extra rules — complete shape, and every parent dominating its children — and the array layout falls out of the shape rule for free, index arithmetic in place of pointers. Insert and extract-max are each a single walk up or down, O(log n); building a heap bottom-up is O(n). Closes with where heaps actually live: priority queues, top-K, and heap sort. 17 June 2026: Binary Search Trees Take Monday's binary tree and add one rule: every value in the left subtree is less than the node, every value in the right is greater. The rule pays off twice — search and insert both follow the same three-case recurrence as binary search, and inorder traversal produces sorted output for free. Closes with a range-query practice problem and the worst-case shape (sorted input degenerates the tree into a linked list). Part 2 — deletion in three cases, inorder successor/predecessor, validate BST, LCA, and recovering a corrupted BST — was held Saturday 20 June, and the unit was capped on 22 June with the two ways to walk a tree, DFS and BFS. The Part 2 deck is linked from the day page. 15 June 2026: Binary Trees Two sessions on binary trees as the first data structure where recursion is the obvious tool — because the structure of the data is recursive. From the linked-list node to the binary-tree node, then traversals as a single skeleton with the work line in three positions (preorder / inorder / postorder), then size, height, and leaf count as the same skeleton with different return values. Closes with three problems: evaluating an expression tree, mirroring a tree, and root-to-leaf path sum. 13 June 2026: Subsets, Merge Sort, Quick Sort Three-hour class. One more backtracking shape — subsets, as a delta from Monday's permute() — then a new recursion family: divide and conquer, through merge sort and quick sort. Time complexity for both sorts in best, average, and worst cases. A short look at iterative subsets and permutations along the way. 8 June 2026: Permutations and Backtracking Post-midsem opens with permutations of {1, 2, 3, 4} built up properly: the recursive insight (same problem, smaller input), the one piece of state (the path so far), and the choose-recurse-undo pattern. Practice list runs to Friday. 4 June 2026: Exam I Coding contest on CodeChef (ES242N26E1) and MCQ quiz on learn.csbasics.in.
