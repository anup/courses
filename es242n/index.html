Times are IST. 19 June 2026: Binary Search Trees (Part 2) The hardest BST operation — deletion — through its three cases: leaf (unlink), one child (promote it), two children (the successor trick). Then validate-BST as the converse of Wednesday's "inorder is sorted" lemma, using the same trap counterexample from Part 1. Closes with three tree problems: lowest common ancestor (the BST property collapses O(n) into O(height)), building a balanced BST from a sorted array, and recovering a corrupted BST by reading its inorder for descending steps. 17 June 2026: Binary Search Trees (Part 1) Take Monday's binary tree and add one rule: every value in the left subtree is less than the node, every value in the right is greater. The rule pays off twice — search and insert both follow the same three-case recurrence as binary search, and inorder traversal produces sorted output for free. Closes with a range-query practice problem and the worst-case shape (sorted input degenerates the tree into a linked list). 15 June 2026: Binary Trees Two sessions on binary trees as the first data structure where recursion is the obvious tool — because the structure of the data is recursive. From the linked-list node to the binary-tree node, then traversals as a single skeleton with the work line in three positions (preorder / inorder / postorder), then size, height, and leaf count as the same skeleton with different return values. Closes with three problems: evaluating an expression tree, mirroring a tree, and root-to-leaf path sum. 13 June 2026: Subsets, Merge Sort, Quick Sort Three-hour class. One more backtracking shape — subsets, as a delta from Monday's permute() — then a new recursion family: divide and conquer, through merge sort and quick sort. Time complexity for both sorts in best, average, and worst cases. A short look at iterative subsets and permutations along the way. 8 June 2026: Permutations and Backtracking Post-midsem opens with permutations of {1, 2, 3, 4} built up properly: the recursive insight (same problem, smaller input), the one piece of state (the path so far), and the choose-recurse-undo pattern.
