ES 242(N): Exam I Curriculum

Foundations of C programming, memory, pointers, arrays, strings, structures, linked lists, time complexity, recursion basics, stacks, and queues.

Exam I

The exam is scheduled in the 3-5 June window. It will test both concepts and C programming.

Concepts

Memory layout, pointer reasoning, data-structure operations, Big-O, and small code-tracing questions.

Programming

Write C programs using arrays, strings, structures, linked lists, recursion basics, stacks, queues, and searching.

1. C Programming Fundamentals

Variables, data types, arithmetic, integer division, conditionals, loops, functions, printf, scanf, format specifiers, and common mistakes in first C programs.

  1. Write and compile a C program using gcc.
  2. Read input using scanf and print formatted output.
  3. Use loops and conditionals to solve simple counting, parity, maximum/minimum, and digit problems.
  4. Use functions to split a program into smaller parts.

2. Arrays in C

One-dimensional arrays, indexing, traversal, passing arrays to functions, insertion, deletion, reversing, copying, sorting by nested loops, and keeping array size separately from array capacity.

  1. Read n integers and compute sum, minimum, maximum, and frequency counts.
  2. Delete an element from an array without leaving a gap.
  3. Insert an element into a position when there is enough capacity.
  4. Reverse an array in place using two indices.

3. 2D Arrays

Row-major order, nested loops, matrix traversal, passing true 2D arrays to functions, and why int a[][m] is not the same as int **.

  1. Read and print a matrix.
  2. Compute row sums, column sums, diagonal sums, and transpose.
  3. Pass a 2D array to a function with the correct column size.
  4. Trace address calculations in row-major order.
Course slides Class page Neso Academy arrays LeetCode 1672 LeetCode 867 LeetCode 1572

4. Pointers and Dynamic Memory

Address-of, dereferencing, pointer types, pointer arithmetic, arrays and pointers, double pointers, dynamic allocation with malloc, calloc, realloc, and releasing memory with free.

  1. Write swap(int *a, int *b).
  2. Return minimum and maximum through pointer parameters.
  3. Allocate a dynamic array, fill it, print it, and free it.
  4. Explain wild pointers, dangling pointers, and memory leaks.

5. Function Pointers, qsort, and bsearch

Function-pointer syntax, passing functions as arguments, comparator functions, sorting arrays with qsort, and searching sorted arrays with bsearch.

  1. Write integer comparators for ascending and descending order.
  2. Sort an array of integers using qsort.
  3. Sort an array of strings or structures using a comparator.
  4. Use bsearch only after the array is sorted in the same order.
Course slides Class page mycodeschool pointers LeetCode 704 LeetCode 912

6. Structures and Unions

Defining structures, grouping related values, arrays of structures, passing structures to functions, pointers to structures, nested structures, structure padding intuition, and basic union behavior.

  1. Store student records as struct Student.
  2. Sort records by marks descending and roll number ascending.
  3. Search records by roll number after sorting.
  4. Use sizeof instead of guessing structure size in real code.

7. Strings in C

The char type, ASCII, character arrays, null termination, string literals, reading strings safely, scansets, buffer overflow, and string functions such as strlen, strcmp, strcpy, and strcat.

  1. Reverse a string in place.
  2. Write your own strlen and strcmp.
  3. Count character frequencies.
  4. Read a full line safely into a fixed-size character array.

8. Linked Lists

Array-based linked lists using data[], nxt[], head, and avail; pointer-based linked lists using struct Node, malloc, and node->next.

  1. Create, traverse, insert at front, insert at end, and delete from a singly linked list.
  2. Free every node in a list without losing the next pointer.
  3. Reverse a linked list iteratively.
  4. Detect a cycle using slow and fast pointers.

9. Time Complexity and Searching

Big-O as growth rate, constants and lower-order terms, linear search, binary search on sorted arrays, nested loops, and comparing two implementations of the same problem.

  1. Identify O(1), O(n), O(n^2), and O(log n) code patterns.
  2. Trace linear search and binary search by hand.
  3. Explain why binary search requires sorted input.
  4. Compare the array and linked-list versions of the Josephus problem.

10. Recursion Basics

Base case, recursive case, trusting the recursive call, simple recursion on numbers and arrays, recursive linked-list traversal, and recursive binary search. Recursion-tree analysis is not part of Exam I.

  1. Write factorial, sum of first n numbers, and array sum recursively.
  2. Print a linked list recursively.
  3. Count linked-list nodes recursively.
  4. Write recursive binary search and explain the base cases.

11. Stacks

Stack ADT, LIFO order, push, pop, peek, array implementation, linked-list implementation, and using a stack to model undo/backtracking-style behavior.

  1. Implement a stack using an array.
  2. Implement a stack using a linked list.
  3. Check balanced parentheses.
  4. Reverse a string using a stack.

12. Queues

Queue ADT, FIFO order, enqueue, dequeue, front, circular array implementation, linked-list implementation, and queue-based level-by-level processing.

  1. Implement a circular queue using an array.
  2. Implement a queue using a linked list with front and rear pointers.
  3. Generate binary numbers from 1 to n using a queue.
  4. Explain the difference between stack order and queue order.
Course slides mycodeschool data structures LeetCode 225 LeetCode 232

Not in Exam I

  • Merge sort and quicksort as formal sorting algorithms.
  • Recursion-tree analysis.
  • Trees, heaps, binary search trees, and hashing.
  • Graphs, BFS, DFS, and graph applications.

Practice strategy

  1. First revise the course slides for the topic.
  2. Then write the basic implementation yourself without looking.
  3. Compile with gcc, test small cases, and test edge cases.
  4. After that, attempt the linked practice problems.
