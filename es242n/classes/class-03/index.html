LeetCode Array Practice

These are array problems selected from the larger LeetCode list. Try them in this order. For each problem, first write the idea in plain English, then write the C solution.

01

LeetCode 1979: Find Greatest Common Divisor of Array

Practice looping over an array and keeping one answer as you scan. Useful if you are still getting comfortable with arrays and helper functions.

02

LeetCode 485: Max Consecutive Ones

Practice a counter that resets when the pattern breaks. This is the same kind of thinking needed for many simple scan problems.

03

LeetCode 283: Move Zeroes

Practice modifying an array in-place. This is harder than just computing an answer, so trace your code on paper before submitting.

04

LeetCode 268: Missing Number

Practice using array values to reason about what is missing. You can solve this in more than one way; start with the approach that is clearest to you.

05

LeetCode 121: Best Time to Buy and Sell Stock

Practice keeping the best answer seen so far while scanning from left to right. This is an important pattern for later array problems.

More Practice

If the first five are comfortable, try these next:

  • LeetCode 1752: Check if Array Is Sorted and Rotated: careful loop logic.
  • LeetCode 88: Merge Sorted Array: two sorted arrays and in-place updates.

When LeetCode shows int* nums, treat nums like an array and use nums[i] to access the element at index i.
