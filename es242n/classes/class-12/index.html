13 June 2026: Subsets, Merge Sort, Quick Sort

Three-hour class. One more backtracking shape (subsets), then a new recursion family — divide and conquer — through merge sort and quick sort.

Slides

B1

Block 1 — Subsets (~40 min)

Subsets on the choose-recurse-undo template — presented as a delta from Monday's permute(): the loop becomes two explicit calls (skip / include), used[] disappears, size replaces depth for printing. Then a short look at iterative subsets and iterative permutations — both are constructive, so iteration also works.

B2

Block 2 — Merge sort (~60 min)

Starts with a problem: two sorted arrays, produce one sorted array, one pass. Invent the merge. Then the recursive shell is three lines of trust-the-call. Time complexity derived level-by-level: O(n log n) in best, average, and worst.

B3

Block 3 — Quick sort (~50 min)

Same divide-and-conquer family with the work in the opposite place: partition before the recursion instead of merge after. The partition contract — pivot in its final position, smaller-on-left, larger-on-right — given as an exercise to design and implement. Then time complexity (best/average O(n log n), worst O(n²)) and pivot-selection strategies.

In-class exercises

  • Write subsets() in C; run on {1, 2, 3, 4}; then break size-- on purpose to see what changes.
  • Solve the merge problem cold: two sorted arrays into one, single pass.
  • Implement merge() + mergeSort() in C; run on [5, 2, 8, 1, 9, 3, 7, 4]; print the array after each merge call to watch it assemble.
  • Design and implement partition(arr, lo, hi) with the last element as pivot; verify the pivot's final index on three test inputs.

Merge, partition, and the two algorithms that use them — plus a few problems that build on the same ideas in disguise.
