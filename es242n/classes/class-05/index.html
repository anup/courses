01

Diagnostic practice and quiz review

Start by revisiting the diagnostic-style questions and the coding problems that students are still debugging. Use this to settle syntax, pointer, array, and 2D-array issues before adding new C features.

02

Function pointers

Introduce function-pointer syntax through small examples, then connect it to qsort comparators. Keep the focus on reading the type and writing one correct comparator.

03

Structures and unions

Use structures to group related values, pass records to functions, and sort arrays of records. Treat structure size carefully: use the slide assumptions for hand calculations, and use sizeof in real code.

04

Strings in C, if time permits

If the class reaches this point, introduce character arrays, the terminating '\0', reading strings safely, strlen, strcmp, and why strings are not compared with ==.

05

Practice briefing

End by showing which problems match arrays, 2D arrays, strings, and record-style C programming. Students should leave knowing what to attempt next and what evidence counts for practice credit.

Practice After This Day

Use this list to continue from the material covered so far. The diagnostic contest will be revisited through quiz-style review in class.

A

LeetCode sorting and binary search

These problems do not force function pointers, but in C the sorting ones are good places to practice qsort and comparator functions.

  • LeetCode 704: Binary Search
  • LeetCode 35: Search Insert Position
  • LeetCode 912: Sort an Array — solve in C using qsort.
  • LeetCode 274: H-Index — sort first, then scan.
B

C structure exercises

Use structures to store records cleanly. For names, assume there are no spaces.

  • Top scorer: read n, then n lines of name roll marks. Print the student with highest marks; if tied, choose the smaller roll number.
  • Sort students: read the same input and print all students sorted by marks descending, then roll number ascending. Use qsort.
  • Find a student by roll number: sort student records by roll number, then use bsearch to find and print the matching record.
  • Sort dates: read n, then n dates as dd mm yyyy. Store each date in a structure and print dates in chronological order.
