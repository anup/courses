24 June 2026: Graphs I — vocabulary, representation, traversal

A graph is the traversal you already know — plus two bolts: how to store the edges, and how not to walk in circles. Session 2 runs BFS and DFS with visited tracking; Friday returns to apply them to cycle detection, connected components, and shortest paths.

Warm-up (≤15 min) — heaps with pointers

One focused insight from the supplementary deck: walk to position k in a binary heap by reading k’s binary representation. Drop the leading 1; the remaining bits spell the route (0 = left, 1 = right). Trace position 6 = 110 → R, L, two steps. Skip insert and extract — those are wrappers around NODE-AT. Hard 15-minute cap; not on any quiz; a closer to Monday’s heap thread, not a graph prelude.

S1

Session 1 — The two pillars, then how to store a graph

Open with a small graph on the board (A through H, ten edges) and ask the class for the fewest hops from A to H. They fan out in rings — that’s BFS, and they already own it from grids. Use the moment to name the two pillars the unit hangs off: representation (we have to write down who connects to whom — a grid’s neighbours are implicit; a graph’s are not) and the visited set (the A–B–C triangle loops forever without one). Frame the unit: graph = traversal + two bolts.

Then build the two representations together on the running graph: adjacency matrix (8×8 of 0/1) and adjacency list (eight rows of neighbour pointers). Trade-off table: space O(V²) vs O(V+E); edge query O(1) vs O(deg). Close with a quick vocabulary pass — vertex, edge, neighbour, degree, path, cycle, connected.

S2

Session 2 — BFS and DFS, with the visited bolt

Animate BFS on the running graph layer by layer (the rings the students just discovered, formalised with a queue). Then run it without the visited set on the A–B–C triangle and watch it loop — earning the bolt the hard way. Pseudocode that handles both queue ordering and visited tracking.

Then DFS — same algorithm with the container swapped (queue → stack, or just recursion). Both visit every vertex once; both are O(V+E). The difference is the visiting order, not the algorithm itself. End on the question Friday answers: what can BFS and DFS decide about a graph — connectivity, cycle, shape?

In-class exercises

  • For the running graph from class, write the adjacency matrix and the adjacency list by hand. Compare which is faster to answer "is A adjacent to D?" vs "list all of B’s neighbours."
  • Trace BFS from A. Write the dequeue order and which vertex first discovers each new one.
  • Trace DFS from A (recursive form). Compare its visiting order with BFS — same vertices, different order?
  • Run BFS without a visited set on the A–B–C triangle. How many times is A revisited in the first 10 dequeues? Where do you stop?
