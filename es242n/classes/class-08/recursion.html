Part 1

The Pattern

Sum this list.

sum on a list

sum_list(list):
   if list is empty: return 0
   return value(head) + sum_list(rest of list) 
int sum_list(Node *h) {
    if (h == NULL) return 0;
    return h->val + sum_list(h->next);
}

A binary tree.

A node is a value and two subtrees. Each subtree is itself a tree — empty, or another node with its own two subtrees.

From any node you see only its value and these two subtrees. You can call the function on a subtree; you cannot peek inside.

  1. Count the nodes.
  2. Sum the values.
  3. Find the maximum.

count bubbles up from the leaves.

Each leaf returns 1. The parent adds 1 (itself) and the two child counts.

count

If count works on the subtrees — what is the count of the whole tree?

count(tree):
   if tree is empty: return 0
   return 1 + count(left subtree) + count(right subtree) 
int count(Node *t) {
    if (t == NULL) return 0;
    return 1 + count(t->left) + count(t->right);
}

sum bubbles up from the leaves.

Each leaf returns its own value. The parent adds its value and the two child sums.

sum

If sum works on the subtrees — what is the sum of the whole tree?

sum(tree):
   if tree is empty: return 0
   return value(root) + sum(left subtree) + sum(right subtree) 
int sum(Node *t) {
    if (t == NULL) return 0;
    return t->val + sum(t->left) + sum(t->right);
}

max bubbles up from the leaves.

Each leaf returns its own value. The parent takes the larger of itself and the two child maxes.

max

If max works on the subtrees — what is the max of the whole tree?

max(tree):
   if tree is empty: return −∞
   return largest of: value(root), max(left subtree), max(right subtree) 
int max(Node *t) {
    if (t == NULL) return INT_MIN;
    int l = max(t->left), r = max(t->right);
    int m = t->val;
    if (l > m) m = l;
    if (r > m) m = r;
    return m;
}

One shape. Three combiners.

count

return 1
+ count(l)
+ count(r);

sum

return val
+ sum(l)
+ sum(r);

max

return max of
val, max(l), max(r);

Trace sum on the tree above.

  • sum(1) = 1 + 0 + 0 = 1
  • sum(5) = 5 + 0 + 0 = 5
  • sum(3) = 3 + 1 + 5 = 9
  • sum(8) = 8,   sum(12) = 12,   sum(9) = 9 + 8 + 12 = 29
  • sum(7) = 7 + 9 + 29 = 45

Trace it once. Then trust the call.

The same template just answered three questions.

What else could you ask about the same tree?

  • Height of the tree
  • Does it contain value x?
  • Count of leaves only
  • Mirror the tree

All of them follow the same template. The structure of the data dictated the structure of the code.

Print every file under this directory.

walk a directory

walk(path, depth):
   if path is not a directory: return

The C version uses opendir / readdir / closedir. The recursion is the same shape — only the boilerplate around it changes.

Evaluate this expression.

eval bubbles values up from the leaves.

Each subtree evaluates itself. The parent applies its operator to the two child values.

eval an expression tree

eval(tree):
   if tree is a leaf: return its value
   a ← eval(left subtree)
   b ← eval(right subtree)
   return apply tree's operator to (a, b) 
int eval(Node *t) {
    if (t->is_leaf) return t->val;
    int a = eval(t->l), b = eval(t->r);
    switch (t->op) {
        case '+': return a + b;
        case '-': return a - b;
        case '*': return a * b;
        case '/': return a / b;
    }
    return 0;
}

Part 2

Monday's Josephus

Monday — three implementations.

// array version, ~20 lines
int josephus_array(int n, int k) {
    int alive[n];
    for (int i = 0; i < n; i++) alive[i] = 1;
    int remaining = n, idx = -1;
    while (remaining > 1) {
        int c = 0;
        while (c < k) {
            idx = (idx + 1) % n;
            if (alive[idx]) c++;
        }
        alive[idx] = 0;
        remaining--;
    }
    for (int i = 0; i < n; i++) if (alive[i]) return i + 1;
    return -1;
}

Why does it shrink?

Suppose you know the survivor of an (n − 1)-person game. Call their position s.

For n people:

  1. Eliminate the k-th person first.
  2. Now n − 1 people remain in a circle.
  3. The next count starts k positions later than the original start.

So the n-survivor sits k positions after the (n − 1)-survivor.
mod n wraps around the circle.

josephus(n, k)  =  (josephus(n − 1, k) + k) mod n

The function behind it.

draw(x, y, length, angle, depth):
   if depth is 0: return
   (x', y') ← endpoint of line from (x, y), given angle and length
   draw line from (x, y) to (x', y')
   draw(x', y', length × 0.72, angle − 30°, depth − 1)
   draw(x', y', length × 0.72, angle + 30°, depth − 1) 
void draw(double x, double y, double len, double ang, int depth) {
    if (depth == 0) return;
    double x2 = x + cos(ang) * len;
    double y2 = y + sin(ang) * len;
    line(x, y, x2, y2);
    draw(x2, y2, len * 0.72, ang - PI/6, depth - 1);
    draw(x2, y2, len * 0.72, ang + PI/6, depth - 1);
}

Simple recursive rule. Complex emergent picture.

The image you saw is literally the call graph of the function.

Every branching point is one stack frame deep. Every leaf is a base-case return.

Part 4

The Stack

Mid-call, sum on 1 → 2 → 3

sum(head=1) — waiting on sum(next)
sum(head=2) — waiting on sum(next)
sum(head=3) — waiting on sum(next)
sum(head=NULL) — returns 0

Bottom frame returns first.

The stack unwinds.

sum(head=1) — returns 6
sum(head=2) — returned 5
sum(head=3) — returned 3
sum(head=NULL) — returned 0

Last in, first out.

Recursion gives you LIFO for free.

Every frame holds its local state until the child returns.

Work before the recursive call → output in forward order.

Work after the recursive call → output in reverse order.

That's why print_reverse reverses a list — without a single "reverse" step.

Part 5

When Recursion Is Slow

What fib(5) actually does.

Red = recomputed.

Fibonacci, the natural way.

fib(n):
   if n < 2: return n
   return fib(n − 1) + fib(n − 2) 
int fib(int n) {
    if (n < 2) return n;
    return fib(n - 1) + fib(n - 2);
}

Three lines. And fib(45) takes minutes.

Remember the answers.

fib(n):
   if n < 2: return n
   if memo[n] already computed: return memo[n]
   memo[n] ← fib(n − 1) + fib(n − 2)
   return memo[n] 
int memo[100];   // -1 initially

int fib(int n) {
    if (n < 2) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fib(n - 1) + fib(n - 2);
}

O(2ⁿ) → O(n). One extra line.

P.S.

One more flavour of recursion

Recursion isn't always a function calling itself.

Two functions calling each other is also recursion. Neither calls itself; the chain still shrinks toward a base case.

A caveat.

The example above is a teaching example. In practice, n % 2 answers the question in one operation — the recursive version is strictly worse.

Mutual recursion really earns its place in parsers and grammars:

expression  →  term ('+' term)*
term        →  factor ('*' factor)*
factor      →  NUMBER | '(' expression ')'

A factor can be an expression in parentheses — which contains terms — which contain factors. Three functions, all calling each other. This recursion has no loop equivalent because the data is inherently nested.
