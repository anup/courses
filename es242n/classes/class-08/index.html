27 May 2026: Stacks and Queues

Two linear data structures built on what you already know. Stacks are LIFO; queues are FIFO. Both have array-backed and linked-list-backed implementations, and both show up everywhere — function calls, expression evaluation, BFS, task scheduling.

Slides

  • Stacks — operations, implementations, and applications
  • Queues — operations, circular array, linked list, and applications
S1

Session 1 — Stacks

What a stack is, push/pop/top/isEmpty, array-backed and linked-list-backed implementations, classic applications: balanced parentheses, function calls, undo, expression evaluation.

S2

Session 2 — Queues

What a queue is, enqueue/dequeue/front/isEmpty, array-backed (circular buffer) and linked-list-backed implementations, classic applications: BFS, task scheduling, buffering. If we run long on stacks, queues spill into the start of the next class.

In-class practice

Submit accepted solutions on judge.csbasics.in. Start with the implementation problems; the application problems build on them.

1

Implement a stack — array-backed

Operations: push, pop, top, isEmpty. Fixed capacity. Return / report stack overflow and underflow conditions explicitly.

2

Implement a stack — linked-list-backed

Same operations as above. Push at the head; pop from the head. No capacity limit (apart from malloc failure).

3

Balanced parentheses

Given a string containing ( ) [ ] { }, return whether it is balanced. Push opening brackets; pop and match on closing brackets.

4

Reverse a string using a stack

Push every character onto a stack, then pop them all to build the reversed string. Demonstrates the LIFO property of a stack directly.

5

Implement a queue — array-backed (circular buffer)

Operations: enqueue, dequeue, front, isEmpty. Use front and rear indices that wrap with %. Handle full vs empty distinction explicitly.

6

Implement a queue — linked-list-backed

Same operations. Keep a head (front, dequeue from here) and a tail (rear, enqueue here). Both pointers move; remember to update tail when the queue becomes empty.

7

Reverse a queue using a stack

Dequeue everything into a stack; pop the stack back into the queue. The first element dequeued ends up at the rear; the last ends up at the front. Reversed.

★

Stretch — evaluate a postfix expression

Given a postfix expression like "3 4 + 2 *" (= 14), evaluate it using a stack. Push numbers; on an operator, pop two operands, apply, push the result. The final stack top is the answer.
