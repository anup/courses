S1

Session 1 — Intuition and state

Try permutations on paper first. Discover the choice tree. Name the two pieces of state — chosen[] and used[] — and see why we need both. End with a hand trace of one path.

S2

Session 2 — Code, bugs, and practice

Write permute() together. Trace it. Then break it: what goes wrong without undo? Without used[]? Without the base case? Close with the practice list for Wednesday.

In-class problems

Permutations on paper

Before any code: list every permutation of {A, B, C} on paper. What pattern did your hand follow?

2

Write permute()

Write the recursive function from scratch using chosen[], used[], and depth. Test it on {1, 2, 3} then {1, 2, 3, 4}.

3

Break it on purpose

Comment out the undo line. Run it. What output do you get? Now restore undo and remove the used[i] check — what's wrong this time?

4

Trace permute(0) for n = 3

By hand. At every call, write the values of depth, chosen[], and used[]. Mark which calls hit the base case and print.
