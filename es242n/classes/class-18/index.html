26 June 2026: Graphs II — deciding things about graphs, and Dijkstra

Wednesday gave us BFS, DFS, and a Graph interface we can pass around. Today we use those tools to decide things about a graph: how many separate pieces it has, whether it has a cycle, whether it’s a tree. Then we climb the difficulty: BFS gives shortest paths on unweighted graphs, and Dijkstra extends that to weighted ones — which is the moment Monday’s heap finally cashes in.

S1

Session 1 — Deciding things about a graph

Three questions, all answered by BFS or DFS plus one line of bookkeeping. Connected components: DFS in a loop over unvisited vertices, count the loop iterations. Cycle detection (undirected): DFS that carries the parent vertex; a visited neighbour that isn’t your parent is a back edge — a cycle. Is-it-a-tree: the consolidation problem — connected ∧ acyclic ∧ n−1 edges. Three independent checks that combine into one answer; students see all of this morning’s tools land at once.

Close with islands / flood fill on a 2-D grid — same DFS, the implicit grid edges are the only thing that changes. The grid is just a graph in disguise.

S2

Session 2 — Shortest path, and Dijkstra

Start where BFS left off: the level a vertex sits at is its shortest-path distance from the source. Wire that into a small change — BFS with a dist array — and you have an unweighted shortest-path algorithm. Trace it on the running graph from Wednesday.

Then add edge weights. Why does plain BFS break? Because the next vertex to finalise is the one with the smallest total distance — not necessarily the next one in line. Swap the queue for a priority queue and you have Dijkstra. The min-heap from Monday is doing the work that makes this run in O((V + E) log V) rather than O(V²). End of unit; the heap unit and the graph unit shake hands.

In-class exercises

  • Extend yesterday’s Graph code: add components(g) that returns the number of connected components, and has_cycle(g) that returns true if there’s any cycle.
  • Combine both into is_tree(g). (Bonus: also check edges == n − 1, and reason about whether that’s redundant.)
  • Write bfs_distances(g, src) that returns a dist[] array — the shortest hop-count from src to every reachable vertex.
  • Code Dijkstra against a weighted variant of the Graph interface (each edge carries an int weight). Use a min-heap as the priority queue — yours from Monday, or the language standard library.
