S1

Session 1 — Adjacency list, properly built; then the graph zoo

Open with the adjacency-list code that ran short last class. The picture: an array of linked lists, where each chain is the same linked list students wrote in week 1 — same Node struct, same head-insert in three lines. Walk through add_edge, has_edge, neighbors in C. Tabulate cost vs the matrix: matrix wins edge query; list wins neighbour scan and space on sparse graphs.

Then a real graph vocabulary, visually: directed vs undirected, weighted, complete (with n(n−1)/2 edges), disconnected (islands / components), self-loop, parallel edges / multigraph. Close with three pieces of math: handshake lemma (∑deg = 2|E|), max edges, and the n−1 lower bound for connectivity — which previews trees.

S2

Session 2 — BFS / DFS as decision tools, then Dijkstra

Three questions, all answered by BFS or DFS plus one line of bookkeeping. Connected components: DFS in a loop over unvisited vertices, count the loop iterations. Cycle detection (undirected): DFS that carries the parent vertex; a visited neighbour that isn’t your parent is a back edge — a cycle. Is-it-a-tree: the consolidation problem — connected ∧ acyclic.

Then climb to shortest paths. BFS levels are shortest-hop distances on an unweighted graph — one dist[] array and you have an algorithm. Add edge weights: plain BFS breaks, because the next vertex to finalise is the one with the smallest total distance, not the next one in line. Swap the queue for a min-heap — that’s Dijkstra, in O((V+E) log V). Monday’s heap unit and the graph unit shake hands. Close with a one-slide preview of minimum spanning trees — the definition; the algorithms (Kruskal, Prim) live in DSA II.

In-class — three things to code

  • Adjacency list — the ListGraph struct, list_graph_create, list_add_edge (one direction; call it twice for undirected), list_has_edge, list_neighbors. Re-use the head-insert pattern from week 1’s linked list.
  • BFS on your adjacency list — then add the three highlighted lines that turn it into bfs_distances: allocate dist[], set dist[source] = 0, and write dist[v] = dist[u] + 1 when you discover v.
  • Dijkstra — extend BFS by swapping the FIFO queue for the min-heap from Monday (flip extract_max → extract_min). Add the relaxation: if (dist[u] + w < dist[v]) dist[v] = dist[u] + w; insert(&h, dist[v], v). Skip popped vertices that are already finalised.
