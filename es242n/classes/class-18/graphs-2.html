Adjacency list — the proper build

An array of linked lists. You’ve written the linked list already.

An array of linked lists

  • One slot per vertex.
  • Each slot = head of a chain.
  • Chain = the linked list from week 1.

Costs

  • Head-insert: O(1)
  • Walk neighbours: O(deg)
  • Space: O(V + E)

The struct, in C

typedef struct AdjNode {
    int dest;
    struct AdjNode *next;
} AdjNode;

typedef struct {
    int n;
    AdjNode **head;     // n head pointers, one chain per vertex
} ListGraph;

ListGraph *list_graph_create(int n) {
    ListGraph *g = malloc(sizeof(ListGraph));
    g->n = n;
    g->head = calloc(n, sizeof(AdjNode *));   // heads start NULL
    return g;
}

add_edge — one direction at a time

void list_add_edge(ListGraph *g, int from, int to) {
    // head-insert `to` into from's chain
    AdjNode *node = malloc(sizeof(AdjNode));
    node->dest = to;
    node->next = g->head[from];
    g->head[from] = node;
}

// Undirected? Call it twice:
//   list_add_edge(g, u, v);
//   list_add_edge(g, v, u);
  • One call → one direction.
  • Undirected edge = two calls, args swapped.
  • Directed graphs: skip the second call.

Building it · three calls

Edges A–B, A–C, B–D.

Edge A–B

add_edge(g, 0, 1) → A: B → ∅
add_edge(g, 1, 0) → B: A → ∅

Edge A–C

add_edge(g, 0, 2) → A: C → B → ∅
add_edge(g, 2, 0) → C: A → ∅

Edge B–D

add_edge(g, 1, 3) → B: D → A → ∅
add_edge(g, 3, 1) → D: B → ∅

  • Head-insert is LIFO — newest neighbour first.
  • The graph is the same; iteration order is what you get.

has_edge and neighbors

bool list_has_edge(const ListGraph *g, int u, int v) {
    for (AdjNode *p = g->head[u]; p != NULL; p = p->next)
        if (p->dest == v) return true;
    return false;
}

int list_neighbors(const ListGraph *g, int u, int *out, int cap) {
    int k = 0;
    for (AdjNode *p = g->head[u]; p != NULL && k < cap; p = p->next)
        out[k++] = p->dest;
    return k;
}

List

  • add_edge: O(1)
  • has_edge: O(deg)
  • neighbors: O(deg)
  • Space: O(V + E)

Matrix

  • add_edge: O(1)

BFS — against your adjacency list

void bfs(const Graph *g, int source) {
    int n = graph_vertex_count(g);
    bool *visited = calloc(n, sizeof(bool));
    int *queue = malloc(n * sizeof(int));
    int head = 0, tail = 0;

    visited[source] = true;
    queue[tail++] = source;

    int neigh[n];
    while (head < tail) {
        int u = queue[head++];
        // ... do work at u ...
        int count = graph_neighbors(g, u, neigh, n);
        for (int i = 0; i < count; i++) {
            int v = neigh[i];
            if (!visited[v]) {
                visited[v] = true;
                queue[tail++] = v;
            }
        }
    }
    free(visited); free(queue);
}

DAG — Directed Acyclic Graph

  • Every edge has a direction.
  • Follow the arrows — you never loop back to where you started.

Bipartite — two sets, every edge crosses

Cycle detection

Does this graph have a loop?

Cycle detection — DFS finds a back edge

● tree edges  ·  ┄ back edge → cycle

  • DFS from anywhere.
  • Hit a neighbour already visited? Back edge → cycle.
  • Catch: in undirected graphs, your parent is always “already visited.” That’s the edge you just walked, not a cycle.
  • Fix: carry the parent in the call. A visited neighbour that isn’t your parent — that’s the real back edge.

Cycle detection — pseudocode (reference)

bool dfs_cycle(const Graph *g, int u, int parent, bool *visited) {
    visited[u] = true;
    int neigh[g->n];
    int count = graph_neighbors(g, u, neigh, g->n);
    for (int i = 0; i < count; i++) {
        int v = neigh[i];
        if (!visited[v]) {
            if (dfs_cycle(g, v, u, visited)) return true;
        } else if (v != parent) {
            return true;                  // ← back edge: cycle
        }
    }
    return false;
}

bool has_cycle(const Graph *g) {
    int n = graph_vertex_count(g);
    bool *visited = calloc(n, sizeof(bool));
    for (int v = 0; v < n; v++)
        if (!visited[v])
            if (dfs_cycle(g, v, -1, visited)) { free(visited); return true; }
    free(visited);
    return false;
}

Is this graph a tree?

The consolidation problem — connected ∧ acyclic.

Three equivalent ways to spot a tree

Any two of these imply the third.

Connected

  • Every vertex reaches every other.
  • Exactly one component.

Acyclic

  • No back edges anywhere.
  • DFS never sees a non-parent visited neighbour.

n − 1 edges

  • One fewer → disconnected.
  • One more → a cycle.

Shortest path on an unweighted graph

BFS, but remember the distance.

BFS levels are distances

  • BFS visits vertices in layers (Wednesday).
  • A vertex’s layer index = its shortest hop-count from the source.
  • One extra array — dist[v] — and you have a shortest-path algorithm for unweighted graphs.

Three lines added to your BFS

int *bfs_distances(const Graph *g, int source) {
    int n = graph_vertex_count(g);
    bool *visited = calloc(n, sizeof(bool));
    int *dist = malloc(n * sizeof(int));                       // NEW
    int *queue = malloc(n * sizeof(int));
    int head = 0, tail = 0;

    visited[source] = true;
    dist[source] = 0;                                          // NEW
    queue[tail++] = source;

    int neigh[n];
    while (head < tail) {
        int u = queue[head++];
        int count = graph_neighbors(g, u, neigh, n);
        for (int i = 0; i < count; i++) {
            int v = neigh[i];
            if (!visited[v]) {
                visited[v] = true;
                dist[v] = dist[u] + 1;                         // NEW — one hop further
                queue[tail++] = v;
            }
        }
    }
    free(visited); free(queue);
    return dist;
}

BFS distance from A — final distances

  • Every vertex carries its shortest hop-count from A.
  • H is 3 hops away — the algorithm finds it; you don’t eyeball it.

What if edges have weights?

  • Each edge carries a number: distance, cost, time.
  • Shortest path A → D? Three options:
  • A→B→D = 2 + 3 = 5
  • A→C→D = 5 + 1 = 6
  • A→B→C→D = 2 + 8 + 1 = 11
  • BFS counts hops, not weights — it can’t pick the cheapest.

Storing the weights — one field in the node

typedef struct AdjNode {
    int dest;
    int weight;                        // NEW — edge weight
    struct AdjNode *next;
} AdjNode;

void list_add_edge(ListGraph *g, int from, int to, int weight) {
    AdjNode *node = malloc(sizeof(AdjNode));
    node->dest = to;
    node->weight = weight;             // NEW
    node->next = g->head[from];
    g->head[from] = node;
}

int graph_weight(const Graph *g, int u, int v) {
    for (AdjNode *p = g->head[u]; p != NULL; p = p->next)
        if (p->dest == v) return p->weight;
    return INT_MAX;                    // edge not present
}

Dijkstra

BFS with a priority queue.

Why FIFO breaks · what to use instead

  • BFS uses a FIFO queue → assumes one hop = one unit of distance.
  • With weights, the next vertex to finalise is the one with the smallest total distance — not the next in line.
  • Fix: swap the queue for a min-heap keyed on tentative distance.
  • Always pop the vertex with the smallest distance → finalise it.
  • Same algorithm. One container swap. Monday’s heap finally pays off.

The heap from Monday — flipped to min

  • You wrote insert, extract_max, is_empty on Monday.
  • For Dijkstra: flip the comparison in extract_max → extract_min.
  • Same array, same indexing arithmetic. Only the “who wins” rule changes.
  • Each item now stores (priority, vertex) instead of a single value. Priority drives the ordering.

Min-heap — copy this

typedef struct { int *prio, *val; int size; } MinHeap;

void heap_init(MinHeap *h, int cap) {
    h->prio = malloc(cap * sizeof(int));
    h->val  = malloc(cap * sizeof(int));
    h->size = 0;
}
bool is_empty(MinHeap *h) { return h->size == 0; }

static void swap(MinHeap *h, int i, int j) {
    int tp = h->prio[i]; h->prio[i] = h->prio[j]; h->prio[j] = tp;
    int tv = h->val[i];  h->val[i]  = h->val[j];  h->val[j]  = tv;
}

void insert(MinHeap *h, int p, int v) {
    int i = h->size++;
    h->prio[i] = p;  h->val[i] = v;
    while (i > 0 && h->prio[(i-1)/2] > h->prio[i]) {
        swap(h, i, (i-1)/2);
        i = (i-1)/2;
    }
}

int extract_min(MinHeap *h) {
    int v = h->val[0];
    h->prio[0] = h->prio[--h->size];
    h->val[0]  = h->val[h->size];
    int i = 0;
    while (1) {
        int l = 2*i+1, r = 2*i+2, m = i;
        if (l < h->size && h->prio[l] < h->prio[m]) m = l;
        if (r < h->size && h->prio[r] < h->prio[m]) m = r;
        if (m == i) break;
        swap(h, i, m);  i = m;
    }
    return v;
}

Dijkstra — the steps

  1. Start with dist[source] = 0, all others = ∞.
  2. Put (0, source) into a min-heap, keyed on distance.
  3. While the heap isn’t empty:
    • Pop the vertex u with the smallest distance.
    • If u is already finalised, skip.
    • Otherwise, mark it finalised. For each neighbour v of u:
      • If dist[u] + weight(u, v) < dist[v], update dist[v] and push (dist[v], v) onto the heap.
  • Relaxation: if (dist[u] + w < dist[v]) dist[v] = dist[u] + w — the heart of Dijkstra.
  • Each relaxation tightens our upper bound on the shortest distance to v.
  • Finalising = no more relaxation is possible.

Dijkstra — pseudocode

int *dijkstra(const Graph *g, int source) {
    int n = graph_vertex_count(g);
    int *dist = malloc(n * sizeof(int));
    bool *done = calloc(n, sizeof(bool));
    for (int i = 0; i < n; i++) dist[i] = INT_MAX;
    MinHeap h;
    heap_init(&h, n * n);                     // safe upper bound on heap entries
    dist[source] = 0;
    insert(&h, 0, source);                    // (priority, vertex)
    int neigh[n];
    while (!is_empty(&h)) {
        int u = extract_min(&h);
        if (done[u]) continue;
        done[u] = true;
        int count = graph_neighbors(g, u, neigh, n);
        for (int i = 0; i < count; i++) {
            int v = neigh[i];
            int w = graph_weight(g, u, v);    // edge weight u→v
            if (dist[u] + w < dist[v]) {       // ← relaxation
                dist[v] = dist[u] + w;
                insert(&h, dist[v], v);
            }
        }
    }
    return dist;
}

Dijkstra — trace

  1. Pop A (0). Relax B → 2, C → 5.
  2. Pop B (2). Skip C (2+8 = 10, worse). Relax D → 5.
  3. Pop C (5). Skip D (5+1 = 6, worse than 5).
  4. Pop D (5). Heap empty. Done.
  • Final: dist[A]=0, B=2, C=5, D=5.
  • A→B→D and A→C→D both reach D at 5 — both are correct.

Why O((V + E) log V)?

  • Each vertex finalised at most once → ≤ V pops.
  • Each edge triggers at most one relaxation → ≤ E pushes.
  • Heap push and pop cost O(log V).
  • Total: (V + E) × O(log V) = O((V + E) log V).
  • Without the heap: scan the unfinalised list each time → O(V²). Heap wins on sparse graphs.
  • Where it’s used: GPS, routing protocols, any weighted reachability problem.

Minimum spanning tree — a preview

● MST = {A–B (2), B–D (3), C–D (1)} — total 6.

  • Take a connected, weighted graph.
  • Spanning tree: n − 1 edges, every vertex reachable, no cycle.
  • Minimum = cheapest such subset.
  • Use: cable, chip wiring, cluster connections — “connect everything cheapest.”
  • Algorithms: Kruskal, Prim (DSA II).
