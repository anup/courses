1 June 2026: Recursion

Structural recursion via the linked list, then generative and combinatorial recursion through binary search, Josephus, and permutations. The bridge from Friday: the call stack is the stack you wrote by hand.

Slides

  • Recursion — full deck, both sessions
S1

Session 1 — Structural recursion

Code that follows the shape of the data. Linked list → trees → DFS on a grid. The marking moves from the caller (Friday) to the callee (today). "Trust the call" named explicitly.

S2

Session 2 — Generative & combinatorial recursion

When the smaller problem is not in the data. Binary search as the cleanest generative example; Josephus as Monday's payoff; permutations as the entry to combinatorial bookkeeping (pick / recurse / undo).

In-class problems

Short, focused. Submit accepted solutions on judge.csbasics.in. The full practice list (with LeetCode links) is below.

1

Linked list family

Write recursive countNodes, sumList, findValue, printForward, and printReverse for a singly linked list.

2

reverseRec

Recursive reverse of a linked list. Pointer surgery on the way back up the stack.

3

Recursive linear search

Return the index of the first occurrence of x in a[lo..n-1], or -1 if absent. Base case at lo == n; recursive case at a[lo] == x or recurse on lo + 1.

4

Recursive DFS on a grid

Count connected components of 1s in a 2D grid using recursive DFS. No explicit stack — the call stack does the work.

5

Permutations

Print all permutations of {1, 2, 3, 4}. Pick → recurse → undo.

6

Subsets

Print all subsets of {1, 2, 3}. For each element: include it or skip it, then recurse. Same pick / undo pattern as permutations.

7

Trace josephus(7, 3)

By hand. Build the recurrence table from n = 1 up to n = 7. Verify the answer matches Monday's iterative survivor.

LeetCode practice — structural recursion (linked lists)

Each problem either has a clean recursive solution or is the recursive form of something you've seen iteratively.

  • 206. Reverse Linked List (Easy) — exactly reverseRec from the deck.
  • 21. Merge Two Sorted Lists (Easy) — classic two-list recursion; pick the smaller head, recurse on the rest.
  • 234. Palindrome Linked List (Easy) — comparing nodes on the way back up the stack.
  • 24. Swap Nodes in Pairs (Medium) — pointer rewire pattern, structural with a careful combine.

LeetCode practice — structural recursion (trees)

Trees are where structural recursion really earns its keep — iteration needs an explicit stack you manage by hand.

  • 104. Maximum Depth of Binary Tree (Easy) — three lines. Probably the cleanest tree-recursion problem.
  • 100. Same Tree (Easy) — two recursive calls, one per child pair.
  • 226. Invert Binary Tree (Easy) — swap and recurse.
  • 111. Minimum Depth of Binary Tree (Easy) — careful base case when one child is NULL.
  • 543. Diameter of Binary Tree (Easy) — two answers per call: the height returned, the diameter tracked globally.

LeetCode practice — DFS with recursion (grid)

Friday you wrote these with an explicit stack. Now write them recursively — the call stack does the bookkeeping.

  • 733. Flood Fill (Easy) — same problem you solved Friday, with recursion.
  • 200. Number of Islands (Medium) — count + recursive DFS sink.
  • 695. Max Area of Island (Medium) — DFS returns the size of each region.
  • 130. Surrounded Regions (Medium) — DFS from border cells to mark "safe" regions.

LeetCode practice — generative recursion

The smaller problem is not in the data — you construct it. Binary-search-style.

  • 704. Binary Search (Easy) — write the recursive form even if iteration is fine.
  • 50. Pow(x, n) (Medium) — x^n = x^(n/2) * x^(n/2); halve, recurse, combine.
  • 70. Climbing Stairs (Easy) — Fibonacci recurrence. Try the naive version, see why it's slow on large n.
  • 509. Fibonacci Number (Easy) — explicit Fibonacci. Compare naive vs memoised.
  • 1823. Find the Winner of the Circular Game (Medium) — Josephus, in LeetCode form.

LeetCode practice — combinatorial recursion / backtracking

Same pattern as permutations: pick → recurse → undo. State management is the new skill.

  • 46. Permutations (Medium) — exactly the slide. Print or return all permutations.
  • 78. Subsets (Medium) — for each element: include or skip, then recurse. Two-call branching at each step.
  • 77. Combinations (Medium) — like subsets but only fixed-size k.
  • 17. Letter Combinations of a Phone Number (Medium) — branch by digit's possible letters.
  • 39. Combination Sum (Medium) — pick / recurse / undo with reuse allowed.
  • 79. Word Search (Medium) — backtracking on a grid; combines DFS with combinatorial undo.
  • 22. Generate Parentheses (Medium) — clean backtracking on a constrained alphabet.

Stretch — when you're ready

  • 51. N-Queens (Hard) — the classic backtracking problem; column / diagonal bookkeeping.
  • 37. Sudoku Solver (Hard) — backtracking on a 9×9 grid.
  • 110. Balanced Binary Tree (Easy) — height check with early return.

Exam I — 3 to 5 June

In scope for recursion: identifying the base case, identifying the smaller problem, tracing one or two recursive calls, recognising the call-stack behaviour of simple list-or-array-suffix recursion. Out of scope: full memoisation, exponential-cost analysis, deep tree problems, formal complexity proofs — those come post-mid-sem.
