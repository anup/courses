17 June 2026: Binary Search Trees (Part 1)

Take Monday's binary tree and add one rule about how values are arranged. The rule pays off twice: search and insert both follow the same three-case recurrence as binary search, and inorder traversal produces sorted output for free.

S1

Session 1 — The property, search, insert

Define the BST property (every value in the left subtree less than the node, every value in the right subtree greater) and watch class judge four small trees for it. Spend time on the local-vs-global trap — a tree can satisfy every parent–child comparison and still violate the property at a deeper descendant. Motivate the structure against an unsorted tree (O(n) search) and a sorted array (O(n) insert).

Then the three-case search recurrence (equal, less, greater) in recursive and iterative form, traced on the canonical tree. Insertion as "walk until NULL, plant a new leaf" — recursive return-the-subtree pattern as canonical, iterative version shown alongside. Trace building a tree from a sequence on the board; then show three different orderings of the same five values producing three different trees.

S2

Session 2 — Min / max, inorder = sorted, practice, worst case

Min and max as "walk all the way left / right" — recursive and iterative side by side. Then the headline result: inorder traversal of a BST visits values in sorted order. Inductive argument from the BST property, demo on the tree built in Session 1, and a one-slide preview of kth-smallest via inorder.

Two practice problems applying the day's tools: range query (print values in [lo, hi] using BST pruning), and counting comparisons on a search trace. Close with the worst-case discussion — sorted insertion degenerates the tree into a linked list, search collapses back to O(n), and balanced BSTs (AVL, red-black, not in our scope) exist to fix this.

One-line preview of Part 2 (Saturday 20 June): deletion, validate BST, inorder successor/predecessor, and tree problems.

In-class exercises

  • Judge four small trees as BST or not — including the local-vs-global trap.
  • Trace search(40) and search(25) on the canonical 7-node tree.
  • Build a BST by inserting 50, 30, 70, 20, 40, 60, 80 one value at a time.
  • Try three different orderings of {2, 3, 5, 7, 8} and predict each resulting tree.
  • Write the range-query function with pruning, then trace it on [25, 65].
  • Count comparisons made by search(target) for 50, 80, 45, 100.
