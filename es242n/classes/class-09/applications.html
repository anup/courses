Part 1

Postfix — a stack does the arithmetic

Two ways to write the same expression

FormExampleOperator goes…
Infix(3 + 4) × 2between the operands
Postfix (RPN)3 4 + 2 ×after the operands

Postfix needs no parentheses and no precedence rules. The order of the tokens already says everything.

RPN = Reverse Polish Notation. Calculators and compilers convert infix to postfix internally, because postfix is trivial to evaluate — with a stack.

Answer: the value left on the stack — 14.

The rule

eval_postfix(tokens):
   for each token:
      if token is a number: push it
      if token is an operator:
         b ← pop,   a ← pop
         push (a op b)
   return pop
Order matters for − and /: the first value popped is the right operand. For 5 3 − you pop 3, then 5, and compute 5 − 3, not 3 − 5.

Part 2

Exploring a grid — DFS and BFS

A grid is just a 2D array of cells.

White = open · dark = wall · green = start (S) · orange = target (T). From a cell you may step up, down, left, or right.

One question, two strategies

DFS — go deep

Follow one path as far as it goes. Hit a dead end? Back up and try another branch.

Tool: a stack. Last cell in is the first explored next.

BFS — go wide

Explore all cells one step away, then all cells two steps away, and so on — in waves.

Tool: a queue. First cell in is the first explored next.

Same algorithm skeleton. Swap the stack for a queue and the traversal order changes from "deep" to "wide." That is the entire difference.

DFS — with a stack

dfs(grid, start):
   push start onto a stack
   while stack is not empty:
      cell ← pop
      if cell already visited: skip
      mark cell visited
      for each open neighbour: push it

Pop the most recently pushed cell each time → you dive down one path before exploring siblings.

BFS — with a queue

bfs(grid, start):
   enqueue start
   while queue is not empty:
      cell ← dequeue
      if cell already visited: skip
      mark cell visited
      for each open neighbour: enqueue it

Dequeue the oldest cell each time → you finish everything one step away before going further.

The two functions are identical except for one word: pop (stack) versus dequeue (queue). The container is the algorithm.

Warm-up — Flood Fill

LeetCode 733 — Flood Fill (Easy).
Given a grid of colours and a start cell, recolour the whole connected region of the same colour (like the paint-bucket tool).

Walk outward from the start cell to every same-colour neighbour, recolouring as you go. BFS (queue) or DFS (stack) — both work, because you only need to reach every cell, not reach it quickly.

The classic — Number of Islands

LeetCode 200 — Number of Islands (Medium).
Grid of '1' (land) and '0' (water). Count the connected groups of land.

Scan the grid. Every time you find an unvisited land cell, that's a new island — run a traversal from it to mark the entire island visited, then keep scanning.

Solve it once with a stack (DFS), once with a queue (BFS). Same answer. Notice the code barely changes.

Part 3

When BFS beats DFS — shortest path

Why a queue, not a stack?

BFS explores in waves: all cells 1 step away, then 2 steps, then 3. So the first time BFS reaches the target, it has arrived by the shortest route.

DFS dives down one path first — it might reach the target by a long, winding route before ever trying the short one.

If the question is "can I reach it?", either works. If the question is "what is the shortest way to reach it?", you need BFS — the queue is what makes the waves possible.

Your turn — shortest path

LeetCode 1091 — Shortest Path in Binary Matrix (Medium).
Grid of 0 (open) and 1 (blocked). Find the length of the shortest path from the top-left to the bottom-right, moving in 8 directions.

BFS from the start. Track the distance with each cell as you enqueue it. The first time you dequeue the target, that distance is the answer.

What we did

  • Postfix: a stack evaluates arithmetic with no parentheses — push operands, apply operators by popping two.
  • DFS: a stack explores deep. BFS: a queue explores wide. Same code, different container.
  • Shortest path: BFS's waves mean the first arrival is the shortest — that's why the queue matters.
Pick the container that matches the job. Stack for "go deep" and for "undo the most recent thing." Queue for "fairly, in order" and for "shortest first."
