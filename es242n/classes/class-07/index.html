25 May 2026: Time Complexity

Today's agenda is time complexity. We get there by solving one problem — Josephus — three times: with an array, with pointers over the same array, and with a circular linked list. The logic stays the same; only the data structure changes. At the end, we name what each version cost.

Slides

  • One Problem, Three Implementations — The Josephus Progression — main deck for the day
  • Time Complexity — referenced in the reflection at the end of Session 2
  • Reference solutions — full C code for all six programs (three Josephus, three reverse). Show only after students have attempted.

Existing reference decks if students want to revisit pointer or linked-list mechanics: Pointers I, Pointers II, Pointers III, Linked List I, Linked List II.

S1

Session 1 — Icebreaker, Josephus on array, Josephus on pointers

Icebreaker puzzle (7 people, k=3, who survives?). Solve Josephus on an array using the mark-and-skip approach. Then rewrite the same logic using pointer arithmetic over the same memory. Same algorithm, different syntax.

S2

Session 2 — Josephus on linked list, Big-O reflection, wrap problem

Build a circular linked list and eliminate by unlinking. Compare the three implementations side by side and connect the cost intuition to the formal Big-O families from the time-complexity deck (linear search, binary search, bubble sort). Wrap with one fresh problem solved three ways.

In-class practice

Every student is expected to submit at least one accepted solution in class today.

A

Josephus — array version

Implement josephus_array(n, k). Use a flag array alive[], walk with idx = (idx + 1) % n, and mark eliminated positions as 0. Verify that n = 7, k = 3 produces survivor 4.

B

Josephus — pointer version

Implement josephus_pointer(n, k) using only pointer arithmetic over the same alive[] array. No alive[i] syntax. Walk with cur++ and wrap with if (cur == end) cur = start;. Verify the answer matches the array version.

C

Josephus — circular linked list version

Build a circular linked list using malloc. Walk k - 1 steps each round, rewire prev->next = cur->next, and free the eliminated node. Verify the answer matches the previous two implementations.

R

Reflection — Big-O of each version

For each implementation, count the work done per elimination and across the whole game. Connect the patterns to O(n), O(n²), and O(log n) using the linear search, bubble sort, and binary search examples in the time-complexity deck.

X

Wrap — reverse a sequence three ways

Reverse a sequence of n integers using (1) an array with index-based two-pointer swap, (2) the same swap using pointer arithmetic, and (3) a pointer-based linked list returning a new head. Notice that the linked-list version is the only one that cannot just "swap from both ends."
