Before you read these

These solutions are reference code. If you haven't yet attempted the array, pointer, and linked-list versions yourself, stop and code first. Reading the answer to a problem you haven't fought with is the fastest way to feel like you understood without actually having understood.

Every solution here compiles cleanly with gcc -std=c11 -Wall. Verify with n = 7, k = 3 → 4 and n = 41, k = 3 → 31.

Josephus — array (mark and skip)

#include <stdio.h>

int josephus_array(int n, int k) {
    int alive[n];
    for (int i = 0; i < n; i++) alive[i] = 1;

    int remaining = n;
    int idx = -1;  // first ++ lands on index 0

    while (remaining > 1) {
        int count = 0;
        while (count < k) {
            idx = (idx + 1) % n;
            if (alive[idx]) count++;
        }
        alive[idx] = 0;
        remaining--;
    }

    for (int i = 0; i < n; i++)
        if (alive[i]) return i + 1;   // 1-indexed
    return -1;
}

int main(void) {
    int n, k;
    scanf("%d %d", &n, &k);
    printf("%d\n", josephus_array(n, k));
    return 0;
}

What this version is doing

  • alive[i] = 1 for "still in the circle," 0 for "out."
  • The outer loop runs until one person remains.
  • The inner loop steps the index forward, wrapping with %. Only alive slots count toward k; dead ones are silently skipped.
  • After eliminating, we leave the index sitting on the eliminated slot. The next round's first ++ moves us off it.
VLA (int alive[n]) requires C99 or later. Use gcc -std=c11. If your platform refuses VLAs, fall back to int alive[1000] with a comment noting the bound.

Josephus — pointer arithmetic

#include <stdio.h>

int josephus_pointer(int n, int k) {
    int alive[n];
    int *start = alive;
    int *end   = alive + n;

    for (int *p = start; p < end; p++) *p = 1;

    int remaining = n;
    int *cur = end - 1;  // first cur++ lands on start

    while (remaining > 1) {
        int count = 0;
        while (count < k) {
            cur++;
            if (cur == end) cur = start;
            if (*cur) count++;
        }
        *cur = 0;
        remaining--;
    }

    for (int *p = start; p < end; p++)
        if (*p) return (int)(p - start) + 1;
    return -1;
}

int main(void) {
    int n, k;
    scanf("%d %d", &n, &k);
    printf("%d\n", josephus_pointer(n, k));
    return 0;
}

What changed from the array version

  • No [] anywhere. *cur reads the flag; *cur = 0 writes it.
  • Two anchor pointers: start and end. The circular wrap is now if (cur == end) cur = start;, not %.
  • Survivor's index is cur - start. Pointer subtraction returns the number of elements between, not bytes.
The algorithm is unchanged. We just rewrote "where am I" in the other dialect of C.

Josephus — circular linked list

#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int pos;
    struct Node *next;
} Node;

int josephus_list(int n, int k) {
    Node *head = malloc(sizeof(Node));
    head->pos = 1;
    Node *tail = head;
    for (int i = 2; i <= n; i++) {
        tail->next = malloc(sizeof(Node));
        tail = tail->next;
        tail->pos = i;
    }
    tail->next = head;          // close the loop

    Node *prev = tail;
    Node *cur  = head;

    while (cur->next != cur) {
        for (int s = 1; s < k; s++) { prev = cur; cur = cur->next; }
        prev->next = cur->next;
        free(cur);
        cur = prev->next;
    }

    int survivor = cur->pos;
    free(cur);
    return survivor;
}

int main(void) {
    int n, k;
    scanf("%d %d", &n, &k);
    printf("%d\n", josephus_list(n, k));
    return 0;
}

What changed from the array version

  • No "alive flag." There are no dead nodes — we removed them.
  • Walk k - 1 steps each round. The k-th node is cur; unlink with prev->next = cur->next and free(cur).
  • Loop exit: cur->next == cur. A node whose next points to itself is a one-element circular list.
Every malloc is matched by a free, including the survivor at the end. Quietly correct memory hygiene is worth more than a "no leaks" comment.

Reverse — array, indices

#include <stdio.h>

void reverse_array(int *a, int n) {
    int i = 0, j = n - 1;
    while (i < j) {
        int t = a[i]; a[i] = a[j]; a[j] = t;
        i++;
        j--;
    }
}

int main(void) {
    int n;
    scanf("%d", &n);
    int a[n];
    for (int i = 0; i < n; i++) scanf("%d", &a[i]);

    reverse_array(a, n);

    for (int i = 0; i < n; i++) printf("%d%c", a[i], i == n - 1 ? '\n' : ' ');
    return 0;
}

Two indices walk in from both ends, swap, meet in the middle. The array isn't shifted; only contents move.

Reverse — array, pointer arithmetic

#include <stdio.h>

void reverse_pointer(int *a, int n) {
    int *lo = a;
    int *hi = a + n - 1;
    while (lo < hi) {
        int t = *lo; *lo = *hi; *hi = t;
        lo++;
        hi--;
    }
}

int main(void) {
    int n;
    scanf("%d", &n);
    int a[n];
    for (int *p = a; p < a + n; p++) scanf("%d", p);

    reverse_pointer(a, n);

    for (int *p = a; p < a + n; p++) printf("%d%c", *p, p == a + n - 1 ? '\n' : ' ');
    return 0;
}

Same swap, same number of comparisons. The only difference is that lo and hi hold addresses, and the comparison lo < hi is comparing addresses.

Reverse — singly linked list

#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int val;
    struct Node *next;
} Node;

Node *reverse_list(Node *head) {
    Node *prev = NULL;
    Node *cur  = head;
    while (cur) {
        Node *nxt = cur->next;
        cur->next = prev;
        prev = cur;
        cur  = nxt;
    }
    return prev;   // new head
}

int main(void) {
    int n;
    scanf("%d", &n);

    Node *head = NULL, *tail = NULL;
    for (int i = 0; i < n; i++) {
        Node *node = malloc(sizeof(Node));
        scanf("%d", &node->val);
        node->next = NULL;
        if (!head) head = node;
        else tail->next = node;
        tail = node;
    }

    head = reverse_list(head);

    for (Node *p = head; p; p = p->next)
        printf("%d%c", p->val, p->next ? ' ' : '\n');
    return 0;
}

Why the linked list can't "swap from both ends"

  • There is no random access. You cannot jump to position n - 1 in one step; you'd have to walk there.
  • So instead of swapping, we flip pointers. Each node's next is rewritten to point at what used to be its predecessor.
  • Three pointers do the work: prev (where we came from), cur (where we are), nxt (saved so we don't lose the rest of the list when we overwrite cur->next).
  • When cur falls off the end, prev is the new head.
The data structure shaped the algorithm. The array let us swap; the linked list forced us to walk forward and rewrite pointers as we went.

The point of all six solutions

Six programs, two problems. Within each problem, the algorithm is the same and the data structure changes. Between the two problems, the technique changes (mark-and-skip vs. two-pointer swap vs. pointer flip) but the lesson is the same: the data structure shapes how the algorithm reads.

If two of your three Josephus implementations disagree, the bug is in whichever one disagrees with the other two. Use the agreeing pair as the oracle.
