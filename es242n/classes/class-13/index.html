15 June 2026: Binary Trees

Two sessions. The first data structure where recursion is the obvious tool — because the structure of the data is recursive. From the linked-list node to the binary-tree node, then traversals, then size/height/leaf count as the same skeleton with different return values.

S1

Session 1 — From recursion to trees as data

Bridge from the linked-list node (one pointer) to the binary-tree node (two pointers). The recursive definition: a tree is empty, or a root + left tree + right tree — same shape as the linked-list definition. Build a small tree by hand in C using a newNode() helper. Then the three traversals as a single skeleton with the work line in three positions (preorder / inorder / postorder). Trace exercise on a 6-node tree.

S2

Session 2 — Computing on trees and applying the pattern

Same recursive skeleton, three different return values: size, height, leafCount. The unifying pattern: combine(left answer, right answer, root info). Then three problems to apply the template:

  1. Evaluate an expression tree — postorder gets a purpose. Each traversal turns out to produce a different classical notation (infix / prefix / postfix). The lesson lands: reading the expression and computing its value are different jobs.
  2. Mirror / invert a binary tree — structural recursion at its simplest.
  3. Root-to-leaf path sum — introduces a new pattern: information flowing down the recursion through a parameter (target shrinks as we descend). This is the technique that makes BST search work on Wednesday.

Close with a one-line preview of BSTs (Wednesday) and array-backed trees (heaps, next week).

In-class exercises

  • Build a 6-node tree given as a drawing — in C, using newNode() and explicit pointer assignments.
  • On paper, trace preorder, inorder, postorder on a 6-node tree.
  • Implement size(), height(), and leafCount() in C and run them on a tree you build yourself.
