S1

Session 1 — MST and Union–Find

Open with the MST question on a 6-vertex weighted graph: which 5 edges minimise the total? After students try by eye, walk through Kruskal (sort edges, take next-cheapest that doesn’t form a cycle) with a full trace, then Prim (grow from a seed, cheapest edge leaving the visited set — uses the min-heap from Monday). A 4-step heap walkthrough makes the “stale entries” mechanic concrete. Skip coding the MST algorithms.

Then Union–Find as the data structure Kruskal’s cycle check needs. Start with the cycle-check problem (naive O(V·E)), then the parent-array structure, then what find and union do conceptually, then a visual trace where the trees grow step by step. Code find/union in C only after the data structure is grounded in pictures. Close with path compression — before/after visual showing find(A) rewiring a chain into a fan-out.

S2

Session 2 — Sets & Maps, Hashing, code the hash set

Bridge slide names the data-structure landscape (min-heap, Union–Find, Set, Map) so students don’t conflate them. Then Set and Map as “what we want” — operations and uses, separate from implementation. Show options for implementing them (cost table: unsorted array, sorted array, BST, hash table) — only hash table is O(1).

Hashing: start with the simplest map — an array, when keys are small integers (direct addressing, hash function = identity). Then the squeeze: h(key) → [0, m) for any key. Hash function properties (deterministic, uniform, fast). Two simple hash functions in C: integer modulo and polynomial string hash, with an explanation of why m should be prime (concrete example: m=10, keys 10, 20, 30, … all hit slot 0).

Collisions: chaining (this is what students code) — each slot is a linked list, head-insert from week 1, cost O(1 + α). Open addressing is shown as the alternative strategy with a 3-state tombstone demo (lookup fails without tombstones, succeeds with), but not coded.

Code the hash set in C — struct + hash function, insert / contains (head-insert + chain walk), remove (unlink). Hash map is shown as a one-slide delta (add a value field). Apply: Contains Duplicate (one-pass hash set) and Two Sum (canonical hash map problem) — with walkthroughs showing why brute force is O(n²) and the hashed version is O(n).

Close with a bonus on count sort (visual: input boxes → count cells → sorted output boxes) and an optional slide on radix sort for students to read on their own.

In-class — code the hash set

  • Struct — an array of Node* (one chain per slot), plus a size counter. Pick a prime M (e.g., 1031).
  • hash_key(int key) — return (unsigned)key % M.
  • set_insert(s, key) — check set_contains first; if absent, head-insert a new node into the chain at hash_key(key).
  • set_contains(s, key) — walk the chain at hash_key(key); return true if found.
  • set_remove(s, key) — walk the chain with a Node **link pointer; on match, unlink and free.
  • Apply: Contains Duplicate — walk the array; for each element, check the set, return true on hit, otherwise insert.
  • (Stretch) Hash map & Two Sum — add a value field to the node; for each nums[i], ask the map for target - nums[i]; if found, return that index and i; otherwise store nums[i] → i.
