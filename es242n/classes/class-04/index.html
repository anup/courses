Practice After This Day

Do these in C before moving to larger problems. The goal is to become comfortable reading and writing pointer code, not just recognizing it on slides.

01

Swap using pointers

Write void swap(int *a, int *b). Call it from main and check that the original variables actually change.

02

Minimum and maximum

Write void min_max(int arr[], int n, int *mn, int *mx). Return both answers through pointer arguments.

03

Array scan with pointer movement

Write a sum or count function using a moving pointer instead of arr[i]. Compare it with your usual array-index version.

04

Return a new array

Write a function that uses malloc to create and return a reversed copy of an input array. Free the returned array in main.

05

2D array row sums

Write a function that prints the sum of each row of a 2D array. Pay attention to the column size in the function parameter.

06

2D array column maximum

Write a function that prints the maximum value in each column. This forces you to switch the order of the row and column loops.

07

SADDLE-POINT

Retry the diagnostic SADDLE-POINT problem after the 2D-array exercises. First find the minimum in each row; then check whether it is also the maximum in its column.

LeetCode Follow-up

After the local C exercises, try these problems. They are still array problems, but they make more sense after pointers, in-place updates, and 2D arrays.

  • LeetCode 27: Remove Element: update an array in-place and return the new length.
  • LeetCode 88: Merge Sorted Array: fill an array from the back using index movement.
  • LeetCode 867: Transpose Matrix: practice row and column indexing in a 2D array.

When LeetCode shows int* nums, treat it as an array and use indexing first. Once that works, try rewriting the same logic using pointer movement.
