Time Complexity

Q1. What is the time complexity of the following code in terms of n?

for (int i = 1; i <= n; i = i * 2)
    for (int j = 0; j < n; j++)
        printf("*");
  1. O(n)
  2. O(n log n)
  3. O(n²)
  4. O(log n)

Q2. Consider the following recursive function:

void recursion(int N) {
    if (N == 0) return;
    printf("%d\n", N);
    recursion(N - 1);
}

Time complexity:

  1. O(1)
  2. O(2N)
  3. O(N)
  4. O(log N)

Space complexity:

  1. O(1)
  2. O(log N)
  3. O(2N)
  4. O(N)

Q3. What is the time complexity of the following code?

void calc(int arr[], int N) {
    int j = 0;
    for (int i = 0; i < N; i++) {
        while (j < N && arr[i] < arr[j]) {
            j++;
        }
    }
}
  1. O(log N)
  2. O(N)
  3. O(N · log N)
  4. O(N²)

Linked Lists

Q4. A singly linked list contains the nodes A → B → C → D → NULL, and head is a pointer to node A. After executing head->next = head->next->next;, which statement is correct?

  1. The list becomes A → C → D → NULL; node B is leaked
  2. The list becomes A → C → D → NULL; no memory leak
  3. The list becomes B → C → D → NULL; node A is leaked
  4. The program crashes with a segmentation fault

Q5. A singly linked list maintains both head and tail pointers. Which of the following operations is NOT O(1)?

  1. Insert at head
  2. Insert at tail
  3. Delete from head
  4. Delete from tail

Q6. Consider the following function intended to free every node of a singly linked list:

void destroyList(struct Node *head) {
    while (head != NULL) {
        free(head);
        head = head->next;
    }
}

What is wrong with this code?

  1. The base case is incorrect; should be head == NULL
  2. After free(head), accessing head->next reads freed memory — undefined behavior
  3. The function will crash with a segmentation fault if head is NULL when called
  4. The function correctly frees all nodes

Function Pointers & qsort

Q9. What does the following declaration declare?

int (*funcs[10])(int, int);
  1. An array of 10 functions, each returning an int and taking two int parameters
  2. An array of 10 pointers to functions, each taking two ints and returning an int
  3. A pointer to an array of 10 functions
  4. A function returning an array of 10 ints

Q10. Which qsort comparator sorts an array of struct Student by score in descending order, with ties broken alphabetically by name?

struct Student { char name[100]; int score; };
  1. int cmp(const void *a, const void *b) {
    return ((struct Student*)b)->score - ((struct Student*)a)->score;
}
  2. int cmp(const void *a, const void *b) {
    struct Student *sa = (struct Student*)a;
    struct Student *sb = (struct Student*)b;
    if (sa->score != sb->score)
        return sb->score - sa->score;
    return strcmp(sa->name, sb->name);
}
  3. int cmp(const void *a, const void *b) {
    struct Student *sa = (struct Student*)a;
    struct Student *sb = (struct Student*)b;
    if (sa->score != sb->score)
        return sa->score - sb->score;
    return strcmp(sa->name, sb->name);
}
  4. int cmp(const void *a, const void *b) {
    return ((struct Student*)b)->score - ((struct Student*)a)->score
         + strcmp(((struct Student*)a)->name, ((struct Student*)b)->name);
}

Q11. You want to sort the following array of strings alphabetically:

char *names[] = {"banana", "apple", "cherry"};
qsort(names, 3, sizeof(char *), cmp);

Which definition of cmp works correctly?

  1. Pass strcmp directly as cmp
  2. int cmp(const void *a, const void *b) {
    return strcmp((char *)a, (char *)b);
}
  3. int cmp(const void *a, const void *b) {
    return strcmp(*(char **)a, *(char **)b);
}
  4. int cmp(const void *a, const void *b) {
    return *(char *)a - *(char *)b;
}

Answer Key & Rationale

# Topic Answer Rationale / Trick
1TimeB Outer loop runs log2n times; inner runs n times → O(n log n).
2TimeC, D N recursive calls (time O(N)); N stack frames (space O(N)). Trick: students often answer O(1) for space because no array is allocated.
3TimeB j is initialized outside the outer loop and never reset. It advances at most N times across the entire run → O(N). Trick: nested loop shape suggests O(N²).
4Linked ListsA The pointer to node B is overwritten without freeing B → memory leak. List structure is correct, but B is unreachable.
5Linked ListsD Even with a tail pointer, deleting the tail node in a singly linked list requires traversing from head to find the new tail → O(n).
6Linked ListsB After free(head), the memory is gone — reading head->next is use-after-free (undefined behavior). Fix: save next before freeing. Trick: C is plainly false (the while (head != NULL) condition handles the empty-list case on the very first line); rules itself out on careful reading.
7QueuesB Elements occupy front, front+1, ..., front+size-1 mod 8 = 6, 7, 0, 1.
8QueuesB Stack reverses order (pop yields 5,4,3,2,1). Queue preserves order (FIFO). Final dequeue order = enqueue order = 5,4,3,2,1.
9Function PointersB Right-left walk: funcs → [10] (array of 10) → * (of pointers) → exit parens → (int, int) (to functions taking two ints) → int (returning int). Trick: A misses that C cannot have arrays of functions, only of pointers to functions.
10Function PointersB Primary key score descending (sb->score - sa->score); on tie, secondary key name ascending via strcmp. Trick: C reverses the score direction; D combines criteria nonsensically.
11Function PointersC qsort passes pointers to elements; the elements are char *, so qsort passes char **. The comparator must dereference once before strcmp. Trick: A is a type mismatch with the qsort signature; B casts char ** to char * → garbage; D compares only the first byte.
12StructuresC a (1 byte) + 3 bytes internal padding + b (4 bytes) + c (1 byte) + 3 bytes trailing padding = 12 bytes. Trick: forgetting trailing padding gives 9.
13UnionsA Big-endian places the most-significant byte at the lowest address → c[0] = 0x12. Compile-and-check trap: a student who runs this on x86 or ARM (both little-endian) gets 0x78 and picks B.
142D ArraysB One row apart = 4 ints × 2 bytes/int = 8 bytes. Compile-and-check trap: on a real machine sizeof(int) = 4, so a student who runs (char*)&arr[2][2] - (char*)&arr[1][2] gets 16 and picks D.
152D ArraysA q is int * → q++ advances by 1 × sizeof(int) = 2 bytes. p is int (*)[4] → p++ advances by 1 × sizeof(int[4]) = 8 bytes. Same starting address; type determines the step. Compile-and-check trap: on a real machine sizeof(int) = 4, so the student gets 4 and 16 and picks B.
