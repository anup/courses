Question: You save two files. The OS picks whichever clusters happen to be free. The clusters of a single file may not be next to each other. How does the OS remember which cluster comes after which?

The data is there, but how does the OS know that cluster 4 comes after cluster 1 for essay.txt?

The File Allocation Table (FAT)

The drive keeps a special array — the FAT. One entry per cluster.

FAT[i] = "what cluster comes after cluster i?"

Cluster 0123 4567 891011 12131415
FAT[ ] 0 4 0 6 9 0 10 0 0 EOF EOF 0 0 0 0 0
essay.txt:   FAT[1]=4 → FAT[4]=9 → FAT[9]=EOF     chain: 1 → 4 → 9 → stop
photo.jpg:   FAT[3]=6 → FAT[6]=10 → FAT[10]=EOF   chain: 3 → 6 → 10 → stop
FAT[i] = 0 means cluster i is free. FAT[i] = EOF means last cluster of a file.

But Where Does the Chain Start?

The drive also has a directory — a table of files. Each entry stores the file's name, size, and first cluster.

FilenameSizeFirst Cluster
essay.txt11 KB1
photo.jpg10 KB3

To read essay.txt:

  1. Directory says first cluster = 1
  2. Read cluster 1.  FAT[1] = 4 → next cluster is 4.
  3. Read cluster 4.  FAT[4] = 9 → next cluster is 9.
  4. Read cluster 9.  FAT[9] = EOF → done.
The directory entry is the head. The FAT array is nxt[]. Each cluster's data is data[].

Our Code vs. a Real USB Drive

What we built in classWhat a USB drive does
data[i] — value stored at slot iContents of cluster i on disk
nxt[i] — index of the next slotFAT[i] — number of the next cluster
nxt[i] = -1 — end of listFAT[i] = EOF — last cluster of file
head — starting indexFirst cluster stored in directory entry
One difference: Our code tracks free slots with an avail chain — a second linked list. FAT doesn't do this. Free clusters are just entries where FAT[i] = 0. To find space, the OS scans the array for zeros. No free chain.

Where Is FAT Used?

FAT is not the filesystem on your laptop's internal drive.

  • Windows uses NTFS (since 1993)
  • Linux uses ext4
  • macOS uses APFS (since 2017)

But FAT is still everywhere:

  • Every USB flash drive you have ever plugged in — FAT32
  • Every SD card in a camera, phone, or Raspberry Pi — FAT32
  • Every computer's boot partition (UEFI) — FAT32
FAT is the one format every OS can read and write. That's why portable storage still uses it. When you copy files to a USB drive, you are writing data into a linked list.
