A Real Problem

You're building a leaderboard for a programming contest.

1

Rating: 2847 • Problems: 3456 • Country: IN
2
rahul_codes
Rating: 2654 • Problems: 2987 • Country: IN
3
priya_iitd
Rating: 2543 • Problems: 2456 • Country: IN

Each player has: username, rating, problems solved, contests, country, rank

Bad Approach: Separate Arrays

char usernames[100000][50];
int ratings[100000];
int problems_solved[100000];
int contests[100000];
char countries[100000][3];
int ranks[100000];
Problems
  • Easy to make mistakes (index mismatches)
  • Hard to keep arrays synchronized
  • Want to sort by rating? Must keep 6 arrays in sync!
  • No logical grouping

Sorting? Nightmare!

How do you sort players by rating?

// Bubble sort — need to swap in ALL 6 arrays!
for (int i = 0; i < n-1; i++) {
    for (int j = 0; j < n-i-1; j++) {
        if (ratings[j] < ratings[j+1]) {
            // Swap ratings
            int temp_rating = ratings[j];
            ratings[j] = ratings[j+1];
            ratings[j+1] = temp_rating;

            // Swap usernames
            char temp_name[50];
            strcpy(temp_name, usernames[j]);
            strcpy(usernames[j], usernames[j+1]);
            strcpy(usernames[j+1], temp_name);

            // Swap problems_solved
            int temp_prob = problems_solved[j];
            problems_solved[j] = problems_solved[j+1];
            problems_solved[j+1] = temp_prob;

            // ... 3 more arrays! (contests, countries, ranks)
        }
    }
}

Forgot to swap one array? Data is now corrupt!

Passing Player Data?

void printPlayer(char username[], int rating,
                 int problems, int contests,
                 char country[], int rank) {
    printf("%s: Rating %d\n", username, rating);
}

6 parameters! Gets worse as you add fields (last active, max streak, tier...)

Enter: Structures

struct Player {
    char username[50];
    int rating;
    int problems_solved;
    int contests;
    char country[3];
    int rank;
};
Benefits
  • All related data in one logical unit
  • Clean, organized, scalable
  • Easier to pass around

Creating a Player

struct Player p1;

strcpy(p1.username, "tourist");
p1.rating = 3952;
p1.problems_solved = 5234;
p1.contests = 187;
strcpy(p1.country, "BY");
p1.rank = 1;

printf("%s has rating %d\n", p1.username, p1.rating);

Passing to Functions

void printPlayer(struct Player p) {
    printf("%s (Rating: %d) - %d problems\n",
           p.username, p.rating, p.problems_solved);
}

void printTopCountry(struct Player p) {
    printf("Top %s player: %s\n",
           p.country, p.username);
}

printPlayer(leaderboard[0]);
printTopCountry(leaderboard[0]);

Clean interface, single parameter instead of 6!

Pointers to Structures

struct Player *ptr = &leaderboard[0];

// Two ways to access members:
printf("%d\n", (*ptr).rating);   // Dereference, then access
printf("%d\n", ptr->rating);     // Arrow operator (preferred)

ptr->rating = 4000;  // Update rating

-> is syntactic sugar for (*ptr).

Calculate Strike Rate

float calculateStrikeRate(struct Batsman b) {
    if (b.balls == 0) return 0.0;
    return (b.runs * 100.0) / b.balls;
}

void printScorecard(struct Batsman b) {
    printf("%-20s %3d (%3d) [%dX4, %dX6] SR: %.1f\n",
           b.name, b.runs, b.balls,
           b.fours, b.sixes, b.strike_rate);
}

// Output: Rohit Sharma        85 ( 62) [8X4, 3X6] SR: 137.1
//         Virat Kohli        112 ( 98) [11X4, 2X6] SR: 114.3

The Rules of Padding

Rule 1: Alignment Requirement

Each type must be stored at a memory address divisible by its size:

  • char (1 byte) → any address
  • short (2 bytes) → address divisible by 2
  • int (4 bytes) → address divisible by 4
  • double (8 bytes) → address divisible by 8
Rule 2: Struct Start Address

The struct itself starts at an address aligned to its largest member.

If largest member is double (8 bytes), struct starts at 0, 8, 16, 24, 32...

Rule 3: Padding at the End

Total size must be a multiple of the largest alignment requirement.

This ensures arrays work correctly (each element starts at valid address).

Why Alignment?

CPUs read memory in fixed-size chunks (typically 4 or 8 bytes at a time).

Aligned int at address 4:

Memory: [ _ _ _ _ | int int int int | _ _ _ _ ]

✓ Read in 1 operation

Misaligned int at address 3:

Memory: [ _ _ _ i | nt in t | i _ _ _ ]

✗ Requires 2 reads + bit shifting (slow or crashes!)

Why Padding at the End?

Consider an array:

struct Student arr[3];

Without end padding (hypothetical size = 26 bytes):

arr[0]: offset 0 (cgpa at 16 ✓ aligned)

arr[1]: offset 26 (cgpa at 42 ✗ not divisible by 8)

arr[2]: offset 52 (cgpa at 68 ✗ not divisible by 8)

With end padding (actual size = 32 bytes):

arr[0]: offset 0 (cgpa at 16 ✓)

arr[1]: offset 32 (cgpa at 48 ✓ divisible by 8)

arr[2]: offset 64 (cgpa at 80 ✓ divisible by 8)

Student Struct: Detailed Breakdown

struct Student {
    char initial;     // 1
    int roll_no;      // 4
    char grade;       // 1
    double cgpa;      // 8
    short semester;   // 2
};

Offset | Field | Size | Padding After

0 | initial | 1 | +3 (to align int to 4)

4 | roll_no | 4 | 0

8 | grade | 1 | +7 (to align double to 8)

16 | cgpa | 8 | 0

24 | semester | 2 | +6 (struct size → 32, multiple of 8)

Total: 32 bytes (16 data + 16 padding!)

Sorting Arrays of Structures

Using qsort()

Real Leaderboard Operations

  • Sort by rating (highest first) → Rank 1, 2, 3...
  • Sort by problems solved
  • Sort by country, then by rating within country
  • Find a player by username

C gives us tools: qsort() and bsearch()

qsort() — The Swiss Army Knife

#include <stdlib.h>

void qsort(void *base, size_t nmemb, size_t size,
           int (*compar)(const void *, const void *));
  • base: pointer to array
  • nmemb: number of elements
  • size: size of each element
  • compar: comparison function (you provide this)

Comparison Function

Sort by rating (descending — highest first):

int compareByRating(const void *a, const void *b) {
    struct Player *p1 = (struct Player *)a;
    struct Player *p2 = (struct Player *)b;

    // Descending: higher rating comes first
    return p2->rating - p1->rating;
}
Return value
  • Negative if a < b
  • 0 if a == b
  • Positive if a > b

Sorting the Leaderboard

struct Player leaderboard[100000];
// ... fill with data ...

qsort(leaderboard, 100000, sizeof(struct Player),
      compareByRating);

// Now leaderboard[0] is highest rated
printf("Rank 1: %s (Rating: %d)\n",
       leaderboard[0].username,
       leaderboard[0].rating);

Multiple Sorting Criteria

// Sort by problems solved
int compareByProblems(const void *a, const void *b) {
    struct Player *p1 = (struct Player *)a;
    struct Player *p2 = (struct Player *)b;
    return p2->problems_solved - p1->problems_solved;
}

qsort(leaderboard, 100000, sizeof(struct Player),
      compareByProblems);

Different comparison functions → different sort orders!

Multi-Level Sorting

Sort by country, then by rating within each country:

int compareByCountryThenRating(const void *a, const void *b) {
    struct Player *p1 = (struct Player *)a;
    struct Player *p2 = (struct Player *)b;

    // First compare countries
    int cmp = strcmp(p1->country, p2->country);
    if (cmp != 0) return cmp;

    // Same country? Compare by rating (descending)
    return p2->rating - p1->rating;
}

Useful for "Top player per country" views

Searching Arrays of Structures

Using bsearch()

bsearch() — Binary Search

#include <stdlib.h>

void *bsearch(const void *key, const void *base,
              size_t nmemb, size_t size,
              int (*compar)(const void *, const void *));
Precondition Array must be sorted (same order as comparison function)

Returns: pointer to found element, or NULL

Finding a Player by Username

// Comparison function for usernames
int compareByUsername(const void *a, const void *b) {
    struct Player *p1 = (struct Player *)a;
    struct Player *p2 = (struct Player *)b;
    return strcmp(p1->username, p2->username);
}

// Sort by username first!
qsort(leaderboard, 100000, sizeof(struct Player),
      compareByUsername);

Binary Search

// Create a key to search for
struct Player key;
strcpy(key.username, "tourist");

struct Player *found = bsearch(&key, leaderboard, 100000,
                               sizeof(struct Player),
                               compareByUsername);

if (found != NULL) {
    printf("Found! Rating: %d\n", found->rating);
} else {
    printf("Player not found\n");
}

Self-Referential Structures

Real problem: Browser history

  • Visit irctc.co.in (book train ticket)
  • Click link → visit paytm.com (pay)
  • Click link → visit flipkart.com (shop)
  • How many pages will you visit? We don't know in advance!

Structure Pointing to Itself

struct PageNode {
    char url[200];
    char title[100];
    struct PageNode *prev;     // Back button
    struct PageNode *next;     // Forward button
};

A structure can contain a pointer to its own type

Creating a History Chain

struct PageNode page1, page2, page3;

// Page 1
strcpy(page1.url, "irctc.co.in");
strcpy(page1.title, "IRCTC");
page1.prev = NULL;
page1.next = &page2;

// Page 2
strcpy(page2.url, "paytm.com");
strcpy(page2.title, "Paytm");
page2.prev = &page1;
page2.next = &page3;

// Page 3 (current)
strcpy(page3.url, "flipkart.com");
strcpy(page3.title, "Flipkart");
page3.prev = &page2;
page3.next = NULL;

Unions

Shared Memory for Different Types

A Real Problem: IP Addresses

IPv4 address: 192.168.1.1

  • Sometimes need it as 4 separate bytes (for printing)
  • Sometimes need it as one 32-bit number (for comparison, system calls)
  • Same data, different views!

Union Definition

#include <stdint.h>

union IPv4Address {
    uint32_t addr;              // Whole address (4 bytes)
    struct {
        uint8_t a, b, c, d;     // Individual octets (4 bytes)
    } octets;
};

Like a structure, but all members share the same memory

Union Memory Layout

union IPv4Address ip;

printf("%lu\n", sizeof(ip));  // 4 bytes
Key difference from structures
  • All members share the same memory location
  • Size = size of largest member (both are 4 bytes here)
  • Writing to one member affects the other!

Using Unions: Different Views

union IPv4Address ip;

// Store as 32-bit number (192.168.1.1 in hex)
ip.addr = 0xC0A80101;

// Read as individual octets
printf("%d.%d.%d.%d\n",
       ip.octets.a,    // 192
       ip.octets.b,    // 168
       ip.octets.c,    // 1
       ip.octets.d);   // 1

Writing to addr automatically sets the octets — they share memory!

Practical Use: Address Comparison

union IPv4Address ip1, ip2;

// Parse "192.168.1.100"
ip1.octets.a = 192;
ip1.octets.b = 168;
ip1.octets.c = 1;
ip1.octets.d = 100;

// Parse "192.168.1.200"
ip2.octets.a = 192;
ip2.octets.b = 168;
ip2.octets.c = 1;
ip2.octets.d = 200;

// Compare as single number (fast!)
if (ip1.addr == ip2.addr) {
    printf("Same network\n");
}

Real Use Cases for Unions

  • Network programming: IP addresses, protocol headers
  • Hardware access: Read/write registers as bytes or words
  • Graphics: Color as RGBA bytes or single 32-bit value
  • Variant types: Store one of several types (with a tag)

Quick Quiz!

Test Your Understanding

Question 1: Memory Size

struct A {
    int x;      // 4 bytes
    char y;     // 1 byte
    int z;      // 4 bytes
};

union B {
    int x;      // 4 bytes
    char y;     // 1 byte
    int z;      // 4 bytes
};

What are sizeof(struct A) and sizeof(union B)?

Answer: struct A = 12 bytes (with padding), union B = 4 bytes

Structures store all members sequentially. Unions share memory — size = largest member.

Question 2: Union Behavior

union IPv4Address ip;
ip.addr = 0xC0A80101;  // 192.168.1.1

printf("%d\n", ip.octets.a);

What will be printed?

Answer: 192 (or 1, depending on endianness)

Union members share memory. Writing addr affects octets!

On little-endian systems (most modern CPUs): octets.a = 1, octets.d = 192

Question 3: Self-Referential Structures

Which of these is VALID?

// Option A
struct Node {
    int data;
    struct Node next;  // Node inside Node?
};

// Option B
struct Node {
    int data;
    struct Node *next;  // Pointer to Node
};

Answer: Option B only!

Can't store entire structure inside itself (infinite size!). But a pointer is always fixed size (8 bytes on 64-bit systems).

Question 4: Structure Access

struct Batsman rohit = {"Rohit Sharma", 85, 62, 8, 3, 137.1};
struct Batsman *ptr = &rohit;

// Which are CORRECT ways to access runs?
printf("%d\n", rohit.runs);        // A
printf("%d\n", ptr.runs);          // B
printf("%d\n", (*ptr).runs);       // C
printf("%d\n", ptr->runs);         // D

Answer: A, C, and D are correct!

B is wrong: ptr.runs tries to access member of pointer (not valid).

Use . for structures, -> for pointers to structures.

Question 5: Cricket Scenario

You have an array of 50 cricketers. You want to find the batsman with the highest strike rate. Which approach is best?

A. Sort using qsort() with strike rate comparison, then take first element

B. Use bsearch() to find maximum strike rate

C. Loop through once, track maximum

Answer: C is best!

Finding max: O(n) loop is simpler and faster than O(n log n) sort.

bsearch() only works on sorted arrays and finds a specific value, not maximum.

Summary

  • Structures: Group related data (players, problems, etc.)
  • Memory: Sequential layout, padding for alignment
  • Nested structures: Structures within structures
  • qsort/bsearch: Built-in tools for sorting and searching
  • Self-referential: Foundation for linked lists
  • Unions: Shared memory for variant types
